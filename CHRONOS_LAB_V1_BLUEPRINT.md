**AI 自动科研系统蓝图**

**版本:** v1.0  
**发布日期:** 2026-03-16  
**状态:** 实施中 (目标：12 个月内运行)

---

## 一、核心定位

**一句话定义:**
> Chronos Lab = AI 自动科研系统

**目标:**
- ❌ 不是：AI 辅助科研
- ✅ 而是：AI 主导科研 + 人类监督

**核心循环:**
```
提出假设 → 建立理论 → 生成预测 → 设计实验 → 执行实验 → 同行评审 → 修正理论
```

---

## 二、7 个 AI Agent 架构

### 1. Literature Agent（文献科学家）

**功能:**
- 读取论文 (arXiv, PubMed, Semantic Scholar)
- 提取知识
- 发现研究空白

**输出:**
- 研究地图
- 知识图谱
- 未解决问题列表

**实现优先级:** ⭐⭐⭐⭐⭐ (第一阶段)

---

### 2. Hypothesis Agent（假设生成器）

**功能:**
- 生成可测试假设
- 必须可证伪 (Karl Popper 原则)

**示例假设:**
> H1: 当系统Φ > 0.8 时，出现自我模型概率增加

**要求:**
- 必须包含证伪条件
- 必须可实验验证

**实现优先级:** ⭐⭐⭐⭐⭐ (第一阶段)

---

### 3. Theory Agent（理论构建器）

**功能:**
- 建立数学模型
- 提出理论结构

**输出:**
- 公理
- 方程
- 推论

**示例:**
> C = Φ × I_env × I_self × I_action

**实现优先级:** ⭐⭐⭐⭐⭐ (第一阶段)

---

### 4. Prediction Agent（预测生成器）

**功能:**
- 每个理论必须产生定量预测

**示例预测:**
> P1: 多模态 AI 的 C 值 < 0.3 → 无主观体验报告  
> P2: 具身机器人的 C 值 > 0.5 → 有主观体验报告

**原则:**
- 没有预测 → 理论无效

**实现优先级:** ⭐⭐⭐⭐⭐ (第一阶段)

---

### 5. Experiment Agent（实验设计师）

**功能:**
- 设计实验变量
- 设计对照组
- 确定样本规模

**输出:**
- 实验协议
- 数据采集方案
- 统计分析方法

**示例实验:**
> Test Φ vs Consciousness  
> 实验对象：人类、动物、AI、机器人

**实现优先级:** ⭐⭐⭐⭐ (第二阶段)

---

### 6. Critic Agent（批评者）⭐ 最重要

**功能:**
- 寻找理论漏洞
- 构造反例
- 挑战实验

**原则:**
> "最重要的进步来自批评。" — 科学史

**示例审查:**
- 数学一致性检查
- 物理边界检查
- Goodhart's Law 检查
- 可证伪性检查
- 奥卡姆剃刀检查

**实现优先级:** ⭐⭐⭐⭐⭐ (第一阶段)

**已有脚本:** `scripts/critic_agent.py`

---

### 7. Revision Agent（修正者）

**功能:**
- 整合实验结果
- 整合批评意见
- 更新理论

**输出:**
- 新版本理论 (ITLCT v12 → v13)
- 修订日志
- 致谢审查者

**实现优先级:** ⭐⭐⭐⭐⭐ (第一阶段)

## 三、自动科研循环

```
Hypothesis Agent
    ↓
Theory Agent
    ↓
Prediction Agent
    ↓
Experiment Agent
    ↓
Critic Agent
    ↓
Revision Agent
    ↓
(进入下一轮循环)
```

**这就是自动科学方法。**

---

## 四、AI 同行评审网络

**Chronos Lab 不应该只依赖一个模型。**

**评审成员:**
- 不同 AI 模型 (Grok、Gemini、ChatGPT 等)
- 人类专家
- 社区成员

**类似:** Open Science Framework

**实现优先级:** ⭐⭐⭐⭐ (第二阶段)

---

## 五、理论挑战系统

**每个理论都进入 Theory Challenge Pool。**

**任何人都可以:**
- 提交反例
- 提出挑战
- 复现实验

**如果挑战成功:**
- 理论必须更新
- 挑战者署名
- 公开挑战记录

**实现优先级:** ⭐⭐⭐⭐ (第二阶段)

## 六、数据基础设施

### 1. 科研文献库

**存储:**
- 论文
- 研究报告
- 实验数据

**实现:** arXiv API + PubMed API + Semantic Scholar API

---

### 2. 实验数据库

**记录:**
- 实验结果
- 模型数据
- 统计结果

**实现:** PostgreSQL + Open Science Framework 兼容

---

### 3. 理论数据库

**版本管理:**
- 理论版本
- 修订历史
- 证伪记录

**实现:** Git for Science (类似 Git 的版本控制)

**实现优先级:** ⭐⭐⭐⭐⭐ (第一阶段)
