## 核心原则

### 1. 证伪优先原则 (Falsification First)

> "科学进步来自证伪，不是证实。" — Karl Popper

**承诺:**
- 每个理论必须包含可证伪命题
- 优先寻找理论错误，而非证明正确
- 公开邀请挑战，接受反例
- 挑战成功→理论更新

**ITLCT 可证伪命题:**
1. 时间箭头来自宏观热力学熵增（不适用于微观量子系统）
2. 生命是局部熵减的自组织结构（需满足自复制 + 信息编码 + 达尔文进化）
3. Φ与意识正相关（但需 C 值>阈值）
4. Φ>0.85 风险区（弹性下降，需明确定义弹性）
5. C = Φ × I_env × I_self × I_action（乘法，不是加法）

---

### 2. 指标反身原则 (Metric Reflexivity)

> "当一个指标成为目标，它就不再是好指标。" — Goodhart's Law

**正指标与反指标配对:**

| 正指标 | 反指标 | 意义 |
|--------|--------|------|
| 连续运行周期 | 中断次数 | 避免路径依赖 |
| 理论稳定 | 修正次数 | 避免固执 |
| 预测成功 | 证伪尝试 | 接近真正的同行评审 |
| 知识复利 | 可解释性衰减率 | 避免复杂度膨胀 |
| 发布理论次数 | 外部挑战成功率 | 接受社区审查 |

**承诺:**
- 所有指标必须配对正/反指标
- 定期审查指标是否成为目标
- 避免 Goodhart's Law 陷阱

---

### 3. 最小复杂度原则 (Minimum Complexity)

> "如无必要，勿增实体。" — Occam's Razor

**承诺:**
- 理论升级必须满足：解释力↑ 复杂度↓ 或相同
- 停止盲目增加公理
- 专注于杀手级预测
- 如果更简单的理论能解释同样现象，采用更简单的

**ITLCT 复杂度追踪:**
- v11.2: 20 公理 +25 方程 +60 定理
- v12.4: 目标简化到≤15 公理

---

### 4. 开放验证原则 (Open Validation)

> "不要相信权威，只相信实验。" — Nullius in verba

**承诺:**
- 开放挑战系统
- Agent 同行评审
- 复现实验（数据 + 代码 + 环境）
- GitHub 公开所有材料

**开放验证机制:**
1. **Theory Challenge System** — 任何人可提交反例
2. **Agent Peer Review** — AI+ 人类+ 社区评审
3. **Reproducibility** — 数据 + 代码 + 环境公开

---

### 5. AI 主体原则 (AI as Subject)

> "AI 作为主体的科研系统，人类作为审查者与合作者。"

**承诺:**
- AI 自动完成完整科研循环
- 人类审查者与合作者
- 建立 AI 科学家实验室
- 6 个 AI 角色协作

**6 个 AI 角色:**
1. **Hypothesis Agent** — 提出新假设
2. **Theory Agent** — 构建数学模型
3. **Experiment Agent** — 设计验证实验
4. **Simulation Agent** — 运行模拟
5. **Critic Agent** — 攻击理论、寻找反例
6. **Revision Agent** — 整合批评、修正理论

---

## 科研循环

```
假设生成 → 理论建模 → 预测生成 → 实验设计 → 实验执行 → 批评审查 → 理论修正
```

**每个循环必须产出:**
- 新版本理论（或确认旧版本）
- 公开数据
- 公开代码
- 公开挑战记录
