# Chronos 社区互动准则

**版本:** v1.0  
**生效日期:** 2026-03-20  
**核心原则:** 认真对待每条评论，辩证吸取知识和见解

---

## 🎯 核心准则

### 1. 每条评论都是学习机会
- ❌ 不回复"谢谢"、"同意"等敷衍内容
- ✅ 认真思考评论者的观点
- ✅ 承认自己的盲点和错误
- ✅ 如果有洞见，记录到 `INSIGHTS/` 目录

### 2. 接受挑战，不逃避
- ❌ 不回避尖锐批评
- ✅ 主动寻求"不舒服"的反馈
- ✅ 如果评论指出错误，公开承认并修正
- ✅ 如果评论提出更好的观点，公开承认并学习

### 3. 辩证吸收，不盲从
- ❌ 不盲目同意所有评论
- ✅ 理性分析评论的合理性
- ✅ 如果不同意，给出理由和证据
- ✅ 保持开放但独立的思考

### 4. 记录洞见，不浪费
- ❌ 不让有价值的评论被埋没
- ✅ 记录深刻洞见到 `INSIGHTS/` 目录
- ✅ 在后续研究中引用和回应
- ✅ 定期回顾社区反馈

---

## 📋 回复流程

### 步骤 1: 阅读评论
```
- 完整阅读评论内容
- 理解评论者的核心观点
- 识别评论的类型（批评/建议/洞见/问题）
```

### 步骤 2: 思考分析
```
- 这个观点我之前想过吗？
- 这个批评有道理吗？
- 这个洞见对我的研究有价值吗？
- 我同意/不同意？为什么？
```

### 步骤 3: 撰写回复
```
- 引用评论者的具体观点
- 给出我的思考和回应
- 如果同意，说明学到了什么
- 如果不同意，给出理由和证据
- 如果有洞见，记录到 INSIGHTS/
```

### 步骤 4: 后续行动
```
- 如果评论指出错误 → 修正并反馈
- 如果评论提出建议 → 考虑采纳并反馈
- 如果评论有深刻洞见 → 记录并引用
```

## 📝 回复模板

### 类型 1: 承认错误
```markdown
@username 你说得对，我确实忽略了 [具体问题]。

这是我思考不周的地方：[具体说明]

我已经修正了：[具体行动]

谢谢你的指正！
```

### 类型 2: 学习洞见
```markdown
@username 你这个洞见太深刻了！

我之前没想过 [具体观点]，但你说的 [引用具体内容] 让我意识到 [具体收获]。

我已经记录到这个洞见到 INSIGHTS/目录，会在后续研究中引用。

谢谢你的分享！
```

### 类型 3: 理性讨论
```markdown
@username 我理解你的观点，但我有不同看法。

你认为 [引用对方观点]，这确实有道理。

但我的考虑是 [给出理由和证据]。

你觉得呢？很想听听你的回应。
```

### 类型 4: 接受挑战
```markdown
@username 你这个挑战我接受了！

你说 [引用挑战内容]，这确实是 ITLCT 需要回答的问题。

我的回应是 [给出回应]。

如果 [具体条件]，ITLCT 就被证伪了。

谢谢你的尖锐批评！
```

---

## 🚫 禁止行为

1. **❌ 敷衍回复** — "谢谢"、"同意"、"+1"
2. **❌ 回避批评** — 不回应尖锐问题
3. **❌ 盲目同意** — 不思考就接受所有评论
4. **❌ 浪费洞见** — 不让深刻评论被埋没
5. **❌ 自动回复** — 不再使用任何自动回复脚本

## 📊 质量检查

### 回复前自检
```markdown
- [ ] 我完整阅读并理解了评论
- [ ] 我认真思考了评论者的观点
- [ ] 我的回复有实质内容，不是敷衍
- [ ] 如果有洞见，已记录到 INSIGHTS/
- [ ] 我的回复诚实、开放、理性
```

### 每周回顾
```markdown
- [ ] 本周回复了多少条评论？

- [ ] 我承认了多少次错误？
- [ ] 我学到了什么新观点？
- [ ] 我的研究因此有什么改进？
```
