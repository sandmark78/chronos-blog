## 🔧 自愈机制

### 错误恢复策略

```json
{
  "max_retries": 3,
  "retry_delay_ms": 60000,
  "backoff_multiplier": 2,
  "alert_after_failures": 3
}
```

**执行逻辑：**
1. 第 1 次失败 → 等待 1 分钟后重试
2. 第 2 次失败 → 等待 2 分钟后重试
3. 第 3 次失败 → 等待 4 分钟后重试 + Telegram 告警
4. 连续 3 次失败 → 暂停任务，等待人工审查

---

### 间隔自适应规则

```
如果连续 5 次执行时间 < 50% 超时：
  → 缩短间隔 25% (但不低于最小间隔)

如果连续 3 次执行时间 > 80% 超时：
  → 延长间隔 25% (或增加超时)

如果单次执行超时：
  → 记录日志，不自动调整 (等待模式稳定)
```

**最小间隔限制：**
- 深度研究：1 小时 (防止过载)
- 文献搜索：2 小时 (API 限制)
- 研究日志：3 小时 (保证质量)
