## 🎯 优化方案

### 方案 1: 假设分层 (推荐)

**三层结构:**

```
层级 1: 核心预测 (10-20 个) ⭐⭐⭐⭐⭐
  - 必须可检验
  - 高影响力
  - 短期可验证 (1-3 年)
  - 示例：dS/dt = σ₀ + σ₁·L + σ₂·Φ + σ₃·L×Φ

层级 2: 衍生假设 (100-200 个) ⭐⭐⭐⭐
  - 从核心预测推导
  - 中等影响力
  - 中期可验证 (3-10 年)
  - 示例：AI 架构Φ上限公式

层级 3: 探索性猜想 (剩余) ⭐⭐⭐
  - 高度推测性
  - 长期或无法验证
  - 标记为"探索性"
  - 示例：信息引力效应
```

**预期效果:**
- 核心预测：**15-20 个** (从 1,041 精简)
- 衍生假设：**150-200 个** (合并相关假设)
- 探索性猜想：**~800 个** (标记为低优先级)

### 方案 2: 问题优先级重分类

**当前:** 1,913 个问题，26.4% 高优先级 (505 个)

**问题:** 505 个"高优先级" = 没有优先级

**优化:**

| 优先级 | 当前 | 目标 | 标准 |
|--------|------|------|------|
| P0 (关键) | - | **10-15 个** | arXiv 提交前必须解决 |
| P1 (高) | 505 | **50-80 个** | Phase 2 实验验证 |
| P2 (中) | ~900 | **200-300 个** | 长期研究 |
| P3 (低) | ~500 | **剩余** | 归档/搁置 |

**操作:**
```bash
# 1. 识别 P0 问题 (10-15 个)
# 标准：影响 arXiv 接受度/理论完整性

# 2. 合并相关问题
# 示例：10 个Φ测量问题 → 1 个"Φ测量方法学"问题

# 3. 归档低优先级
# 移动到 problem-database/archived/
```

### 方案 3: 知识卡片整合

**当前:** 891 张卡片

**问题:**
- 重复内容 (同一假设多张卡片)
- 粒度过细 (一张卡片一个公式)
- 检索困难

**优化:**

| 类型 | 当前 | 目标 | 操作 |
|------|------|------|------|
| 核心卡片 | ~100 | **50-70 张** | 合并相关卡片 |
| 实验方案 | ~100 | **50-70 张** | 整合为实验路线图 |
| 衍生内容 | ~691 | **归档** | 移动到 knowledge/archive/ |

**示例整合:**
```
原: DC-188, DC-189, DC-190 (3 张卡片)
  - 信息扩散速率上限
  - 时间体验 - 信息密度标度律
  - 高信息密度区域主观时间更慢

新: "信息 - 时间耦合" (1 张卡片)
  - 统一公式：T_exp ∝ (I_processing)^α
  - 包含 3 个原假设作为子预测
```

**预期:** 891 张 → **300-400 张** (精简 55-65%)

### 方案 4: 思想实验 → 实证转化

**优化:**

| 类别 | 数量 | 操作 |
|------|------|------|

| 理论启发 | ~100 | 保留为理论工具 |
| 纯思辨 | ~200 | 归档/标记为思辨 |

**示例转化:**
```

  ↓ 实证化
实验方案：麻醉期间Φ值连续监测 (DC-392)
```
