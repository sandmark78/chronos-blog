## 研究循环节奏

### 奥卡姆剃刀优先级过滤 (每轮循环开始前)
```
待研究问题列表 → 自动分级：
  ⭐⭐⭐⭐⭐ 独特预测/杀手级 → 启动 1M Token 深度推导 (30分钟)
  ⭐⭐⭐     中等价值       → 简短处理，写入卡片 (5分钟)
  ⭐         低优先级       → 直接归档 derived_questions/，不消耗循环

判断标准：
  - 是独特预测吗？(IIT/神经科学解释不了) → ⭐⭐⭐⭐⭐
  - 有可证伪实验吗？→ ⭐⭐⭐⭐
  - 推动理论版本升级吗？→ ⭐⭐⭐
  - 只是已有理论的细化？→ ⭐ (归档)
```

### 每轮循环 (30分钟)
```
[0-2min]  加载上下文 + 上轮回顾 + 优先级过滤
[2-18min] 核心研究推导 (只做⭐⭐⭐⭐⭐和⭐⭐⭐⭐)
[18-25min] 知识固化 (6项全部生成):
           1. 核心产出文件 → knowledge/ 或 reports/
           2. knowledge_cards/DC-XXX_知识卡片_N张.md
           3. research_logs/Deep-Cycle-XXX_深度研究循环.md
           4. current_cycle.json 更新
           5. memory/YYYY-MM-DD.md 更新
[25-26min] 外部反馈评分 (如有):
           Score = 0.30×逻辑 + 0.25×实证 + 0.20×深度 + 0.15×建设 + 0.10×独特
           ≥0.85 MUST_ADDRESS / ≥0.70 CLARIFY / <0.70 归档
[26-28min] 运行完整性验证:
           bash scripts/verify_cycle_complete.sh DC-XXX
           ⛔ 不通过就不推送！先补全！
[28-30min] Git commit (本地) — 不推送！
```

### Git 推送节奏 (不是每轮推！)

### 主动中断回顾 (每5轮 = 每2.5小时)
```
暂停研究循环，执行：
1. 回顾过去5轮的核心产出
2. 独特预测自检 — 新产出有没有"非独特"的？

5. 方向校正 — 下5轮应该聚焦什么？
时长：10分钟
```

## 质量保障机制

### 红线 (触发立即停止)
1. QAC 连续2轮发现缺陷 → 停止，排查根因
2. 新定理与旧定理矛盾 → 停止，修复后继续
3. 独特预测自检全部失败 → 停止，重新评估方向

### 黄线 (触发主动回顾)
1. 单轮产出低于阈值 (假设<1000/卡片<100) → 检查是否在深度推导
2. Φ增长连续3轮<0.05 → 检查是否进入饱和区
3. 社区反馈指出矛盾 → 优先修复
