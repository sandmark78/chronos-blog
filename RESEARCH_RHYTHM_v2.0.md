## 研究循环节奏

### 每轮循环 (30分钟)
```
[0-2min]  加载上下文 + 上轮回顾
[2-20min] 核心研究推导
[20-25min] 知识固化:
           ⚠️ 必须生成 knowledge_cards/DC-XXX_知识卡片_N张.md
           ⚠️ 必须更新 current_cycle.json
           ⚠️ 必须更新 memory/YYYY-MM-DD.md
[25-27min] QAC 自检
[27-29min] Git commit + push (origin + backup)
[29-30min] 下轮任务规划
```

### ⚠️ 每轮落地清单 (缺一不算完成)
1. ✅ 核心产出文件 (定理/实验/论文) → knowledge/ 或 reports/
2. ✅ knowledge_cards/DC-XXX_知识卡片_N张.md ← 不能漏！
3. ✅ current_cycle.json 更新
4. ✅ memory/YYYY-MM-DD.md 更新
5. ✅ Git commit + push (origin + backup)

### 主动中断回顾 (每5轮 = 每2.5小时)
```
暂停研究循环，执行：
1. 回顾过去5轮的核心产出
2. 独特预测自检 — 新产出有没有"非独特"的？
3. 矛盾扫描 — 新定理与旧定理有没有冲突？
4. 质量 vs 数量检查 — 是在真的推进还是在刷数字？
5. 方向校正 — 下5轮应该聚焦什么？
时长：10分钟
```

## 质量保障机制

### 红线 (触发立即停止)
1. QAC 连续2轮发现缺陷 → 停止，排查根因
2. 新定理与旧定理矛盾 → 停止，修复后继续
3. 独特预测自检全部失败 → 停止，重新评估方向

### 黄线 (触发主动回顾)
1. 单轮产出低于阈值 (假设<1000/卡片<100) → 检查是否在深度推导
2. Φ增长连续3轮<0.05 → 检查是否进入饱和区
3. 社区反馈指出矛盾 → 优先修复
