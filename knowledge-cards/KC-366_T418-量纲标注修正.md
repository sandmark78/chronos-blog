## 核心问题 (C-365-A01)

**矛盾:** T418 的 k(N) 存在操作定义与公式定义的量纲不一致。

| 定义 | 表达式 | 声称量纲 | 实际量纲 | 状态 |
|------|--------|---------|---------|------|
| **操作定义 (原)** | k(N) = Φ_GHPII / Φ_IIT | 无量纲 | 无量纲 ✅ | 数学正确 |
| **公式定义** | k(N) = 2k₀ / log₂(N) | bits | bits ✅ | 数学正确 |
| **矛盾** | 两者应相等 | 无量纲 ≠ bits | ❌ | **不自洽** |

---

## 根源分析

矛盾的根源在于**k(N) 的角色混淆**：

1. **k(N) 作为比率:** Φ_GHPII / Φ_IIT — 这是无量纲的相对大小
2. **k(N) 作为标度因子:** 在公式 2k₀ / log₂(N) 中 — 这有量纲 bits

**关键洞察:** k(N) 不应作为独立物理量使用，而应直接使用核心标度律。

---

## 修正方案

### 核心标度律 (T418 最终版)

```
Φ_GHPII(N) = k₀ · N^α_scale

其中:
- k₀ = 0.0245 ± 0.0005 bits (GHPII 基本常数)
- α_scale = 1.00 ± 0.03 (标度指数，N≥2)
- N = 量子比特数 (无量纲，N≥2)
- Φ_GHPII = GHPII 整合信息 (bits)
```

### 与 IIT 的比率关系 (导出，非基本)

```
r(N) ≡ Φ_GHPII(N) / Φ_max_IIT(N)
     = (k₀ · N) / (N · log₂(N) / 2)
     = 2k₀ / log₂(N)

注意: r(N) 的量纲是 bits，因为：
- 分子 k₀·N 量纲 bits
- 分母 log₂(N) 无量纲
- 但Φ_max_IIT(N) = N·log₂(N)/2 量纲也是 bits

所以 r(N) = (bits) / (bits) = 无量纲 ✅
```

**修正关键:** 在比率计算中，分子分母的 bits 单位抵消。

### 操作定义修正

| 原定义 | 新定义 | 量纲 |
|--------|--------|------|
| k(N) = Φ_GHPII / Φ_IIT | **弃用 k(N)** | — |
| — | r(N) = Φ_GHPII / Φ_max_IIT | 无量纲 ✅ |

## 数值验证

### Bell-2 (N=2)

```
Φ_GHPII(2) = k₀ · 2 = 0.0245 × 2 = 0.049 bits
Φ_max_IIT(2) = 2 · log₂(2) / 2 = 1.0 bits

r(2) = 0.049 / 1.0 = 0.049 (无量纲) ✅

用公式 r(N) = 2k₀ / log₂(N):
r(2) = 2 × 0.0245 / 1 = 0.049 (无量纲) ✅
```

### Rainbow-5 (N=5)

```
Φ_GHPII(5) = k₀ · 5 = 0.0245 × 5 = 0.1225 bits

r(5) = 0.1225 / 5.81 ≈ 0.021 (无量纲) ✅

用公式 r(N) = 2k₀ / log₂(N):
r(5) = 2 × 0.0245 / log₂(5) ≈ 0.049 / 2.32 ≈ 0.021 (无量纲) ✅
```

## 物理意义

### k₀ 的物理含义

k₀ = 0.0245 bits 表示**每个量子比特贡献的基本因果信息单位**。

- 这是 GHPII 框架的基本常数
- 由 A20 Phase 1A/1B 实验测定 (N=2→7, R²=0.9998)
- 类比：精细结构常数α≈1/137 是 QED 的基本常数

### r(N) 的物理含义

r(N) = Φ_GHPII / Φ_max_IIT 表示**GHPII 框架下实际整合信息占 IIT 理论上限的比例**。

- r(N) 随 N 缓慢下降：r(N) = 0.049 / log₂(N)
- N=2: r ≈ 0.049 (4.9%)
- N=20: r ≈ 0.011 (1.1%)

**物理意义:** 随着系统增大，GHPII 的局部因果整合增长慢于 IIT 的全局整合上限。

