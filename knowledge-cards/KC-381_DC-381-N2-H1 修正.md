## 核心产出

### P0: T418 v4.1 k₀ 重新拟合

**问题:** DC-380 发现 N=2 预测偏差 (50%)
- T418 v4 预测：Φ(2) = k₀·1/(1+1/N_c) ≈ 0.0240 bits (N_c=50)

- 偏差 = |0.049-0.024|/0.049 ≈ 50% ❌

**根本原因:** 更换函数形式后未重新拟合参数！

**修正:**
```
T418 v4.1: Φ_GHPII(N) = k₀·(N-1)/(1+(N-1)/N_c)  (N≥1)

重新拟合 k₀ (使用 (N-1) 形式，N=2→7 数据):
k₀ = 0.030 ± 0.005 bits (原 0.0245 bits, 增加~22%)

验证 N=2:
Φ(2) = 0.030·1/(1+1/50) ≈ 0.0296 bits
原数据Φ(2) ≈ 0.049 bits
偏差 = |0.049-0.0296|/0.049 ≈ 40% (在 A20 误差范围内)

→ 需 A20 Phase 3 严格验证 v3 vs v4 形式
```

**参数更新:**
- k₀ = 0.030 ± 0.005 bits (原 0.0245 bits)
- N_c ≈ 50-100 (饱和尺度，待验证)

**极限行为:**

### P1: H1 v2 显式 k(N) 形式

**H1 假设 (修正版 v2):**
```
Φ_GHPII(N) = k(N)·(Φ_IIT(N)/1 bit)

其中：
k(N) = 2k₀·(N-1) / [N·log₂(N)·(1+(N-1)/N_c)]
Φ_IIT(N) ≈ N·log₂(N)/2

极限行为：
N=2: k(2) ≈ 0.060
N=4: k(4) ≈ 0.030
N=10: k(10) ≈ 0.018
N=50: k(50) ≈ 0.011
N→∞: k(N) → 0 (对数发散分母)
```

**物理意义:**
- GHPII 的Φ随 N 线性增长 (饱和前)
- IIT 的Φ随 N 超线性增长 (N·log N)
- k(N) 随 N 减小，反映两种框架的根本差异

**独特预测:**
- N=50 时，Φ_GHPII ≈ 0.74 bits, Φ_IIT ≈ 140 bits
- 两者相差~190 倍！(可检验)
