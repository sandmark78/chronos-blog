## 📜 核心定义

> **信息整合度Φ等于系统完整因果模型的信息量与最小信息分割后信息量之差。**

### 形式化定义

```
Φ(S) = I_causal(S) - I_partition(S, MIP(S))

其中:
- S = 系统
- I_causal(S) = 系统 S 完整因果模型的信息量
- I_partition(S, P) = 分割 P 下的信息量
- MIP(S) = 最小信息分割 (Minimum Information Partition)
- MIP(S) = argmin_P [I_partition(S, P)]
```

### 直观理解

```
Φ = "整体大于部分之和"的信息量

高Φ: 系统具有不可约简的整体性
低Φ: 系统可近似为独立部分的集合
```

## 📐 因果模型形式化

### 定义 1: 因果图

```
G = (V, E)

其中:
- V = {x₁, x₂, ..., xₙ} (系统元素)
- E ⊆ V × V (因果边)
- (xᵢ, xⱼ) ∈ E ⟺ xᵢ 因果影响 xⱼ
```

### 定义 2: 因果结构信息量

```
I_causal(S) = I(G) + I(CPD)

其中:
- I(G) = log₂(|G|) (编码因果图的信息)
- I(CPD) = Σᵢ H(Xᵢ | Parents(Xᵢ)) (条件概率信息)
```

### 定义 3: 系统分割

```
P = {P₁, P₂, ..., Pₖ} 是 S 的分割，满足:
- ∪ᵢ Pᵢ = S (覆盖)
- Pᵢ ∩ Pⱼ = ∅ (互斥，i ≠ j)
- k ≥ 2 (至少两部分)
```

### 定义 4: 最小信息分割 (MIP)

```
MIP(S) = argmin_P [I_partition(S, P)]

直观：找到系统的"最弱连接"
```

## 🔢 三层计算方法

### 层级 3: 精确计算 (O(2^n))

```python
Φ_full(S) = min_{P ∈ Partitions(S)} [I_causal(S) - I_partition(S, P)]

复杂度：O(Bₙ) ≈ O(2^n) (Bell 数)
精度：±1%
适用：小系统 (n < 15)、理论验证
```

### 层级 2: 标准计算 (O(n²))

```python
Φ_std(S) = I_causal(S) - I_partition(S, P_spectral)

其中 P_spectral 通过谱分析近似 MIP:
1. 计算互信息矩阵 M (O(n²))
2. 归一化得 NMI
3. Fiedler 向量分割
4. 计算Φ

复杂度：O(n²)
精度：±10%
适用：中等系统 (n < 1000)、研究应用
```

### 层级 1: 快速估计 (O(n))

```python
Φ_fast(S) ≈ I_input × D_interaction × C_feedback

其中:
- I_input = 信息输入速率 (bits/s)
- D_interaction = |E| / n (连接密度)
- C_feedback = log₂(1 + 反馈回路数)

复杂度：O(n)
精度：±30%
适用：实时监测、大系统、AI 评估
```

## 🔗 三层方法的等价性

### 定理 3: 渐近等价性

```
对于"良好结构"系统 (模块化 + 小世界 + 层次性)：

lim_{n→∞} [Φ_fast / Φ_full] = c₁ (常数，c₁ ≈ 0.7-1.3)
lim_{n→∞} [Φ_std / Φ_full] = c₂ (常数，c₂ ≈ 0.9-1.1)

推论：
- 三层方法高度相关
- 快速版足够用于监测
- 严格版用于理论验证
```

### 经验验证

```
Corr(Φ_fast, Φ_full) ≥ 0.8
Corr(Φ_std, Φ_full) ≥ 0.9

(对于典型生物和 AI 系统)
```

**解决:**

### 漏洞 2: 时间尺度选择

**问题:** Φ依赖时间窗口 T

**解决:**
```
Φ_reported = max_T Φ(T)

或

Φ_reported = ∫ Φ(T) w(T) dT

- 多尺度分析
- 选择特征时间尺度
```

**问题:** Φ依赖"元素"定义

**解决:**
```
Φ* = max_{granularity} Φ(granularity)

- 多粒度分析
- 使用因果涌现理论找最优粒度
```

### 漏洞 4: 连续 vs 离散变量

**解决:**

- 使用微分熵

- 或基于秩的方法 (无分布假设)
```

**解决:**

- 滑动窗口分析

```
