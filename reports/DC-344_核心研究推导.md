## 二、C01 修正：N_qubits=1 边界 Bug 修复

### 问题描述

DC-343 代码 (`compute_mip_phi` 函数):
```python
split_point = np.random.randint(1, n_qubits)  # ⚠️ n_qubits=1 时崩溃
```

当 `n_qubits = 1`:
- `np.random.randint(1, 1)` → `ValueError: low >= high`

### 修正方案

**理论依据:**
- 单量子比特系统无法进行 MIP 划分 (MIP 要求至少 2 个子系统)
- IIT 4.0 定义：Φ衡量系统的**不可简化性**，单元素系统无"划分"概念
- 因此 Φ(N=1) = 0 (或定义为 NaN，但 0 更直观)

**修正代码:**

```python
def compute_mip_phi(circuit, statevector, n_partitions=10):
    """
    计算最小信息划分Φ (MIP) — DC-344 修正版
    
    参数:
        circuit: QuantumCircuit
        statevector: Statevector
        n_partitions: 采样划分数
    
    返回:
        float: Φ值 (bits)
    """
    n_qubits = circuit.num_qubits
    
    # C01 修正：边界条件处理
    if n_qubits < 2:
        return 0.0  # 单量子比特无 MIP 定义，Φ=0
    
    phi_values = []
    
    # 采样不同的子系统划分
    for _ in range(n_partitions):
        # 随机划分量子比特为两个子系统
        indices = np.random.permutation(n_qubits)
        split_point = np.random.randint(1, n_qubits)  # ✅ 现在 n_qubits >= 2
        subsystem_a = indices[:split_point].tolist()
        subsystem_b = indices[split_point:].tolist()
        
        # ... (后续代码不变)
    
    # MIP: 取最小值
    phi_mip = min(phi_values) if phi_values else 0.0
    
    return phi_mip
```

## 三、C02 修正：GHPII vs IIT 4.0 框架关系澄清

### 问题描述

DC-342 (T416/T417) 基于 **GHPII 全息对应**:
$$\Phi_{GHPII} = \frac{S_{boundary} - S_{bulk}}{S_{max}}$$

DC-343 (Phase 1B) 采用 **IIT 4.0 因果结构**:
$$\Phi_{IIT} = \min_{MIP} \frac{IE}{N_{bits}}$$

**关键矛盾:** 两者物理基础不同，但 DC-343 直接沿用了 DC-342 的 T417 公式，未论证兼容性。

### 框架关系论证

#### 1. 两者本质是**互补的操作定义**

| 维度 | GHPII | IIT 4.0 |
|------|-------|---------|
| **理论基础** | 全息原理 (AdS/CFT 类比) | 整合信息论 (因果结构) |
| **Φ物理意义** | 边界 - 体纠缠差异 | 系统不可简化性 |
| **适用范围** | 量子引力/宇宙学尺度 | 量子信息/神经科学尺度 |
| **数学形式** | 熵差归一化 | MIP 最小信息划分 |
| **计算复杂度** | O(1) (解析) | O(2^N) (数值) |

**核心洞察:** GHPII 和 IIT 4.0 是**同一物理实在的不同表征**:
- GHPII: 从**时空几何**角度描述意识的量子基础
- IIT 4.0: 从**信息因果**角度描述意识的量子基础

#### 2. 对应关系假设 (H1)

**假设 H1:** GHPIIΦ与 IIT 4.0Φ在适当条件下线性相关:
$$\Phi_{GHPII} = k \cdot \Phi_{IIT} + C$$

其中:
- $k$: 比例因子 (依赖系统细节)
- $C$: 背景项 (可能为 0)

**支持证据:**
1. 两者都衡量"量子纠缠/关联的不可简化性"
2. 两者都在 N→1 时→0 (单元素无结构)
3. 两者都随系统复杂度增加而增加

**待验证:** 需要数值模拟验证线性相关性 (DC-344 实验 1)

#### 3. 实验验证策略

DC-344 采用**双框架并行验证**:

**策略 A: GHPII 框架 (理论预测)**
- 使用 T416/T417 公式进行理论预测
- 预测α耦合常数：$\alpha = 0.01 \pm 0.005$
- 预测Φ阈值：$\Phi_{threshold} = \alpha \cdot \ln(\Gamma_{dec}/\Gamma_{sys} + C)$

**策略 B: IIT 4.0 框架 (实验测量)**
- 使用 Phase 1B 代码计算$\Phi_{IIT}$
- 实验拟合$R \propto \Phi_{IIT}^{\alpha_{IIT}}$
- 比较$\alpha_{IIT}$与理论预测$\alpha$

**成功标准:**
- 若$|\alpha_{IIT} - \alpha| / \alpha < 0.2$ → 支持 H1 (框架兼容)
- 若差异>20% → 需修正 H1 或重新推导

#### 4. 框架适用范围声明

**DC-344 明确声明:**

| 任务 | 使用框架 | 理由 |
|------|----------|------|
| **理论推导** (T416/T417) | GHPII | 第一性原理，解析可解 |
| **实验测量** (A20 Phase 1B) | IIT 4.0 | 操作定义明确，可数值计算 |
| **框架验证** (A20 Phase 2) | 双框架对比 | 检验 H1 对应假设 |
| **最终统一** (ITLCT v25.0) | TBD | 待 A20 实验结果决定 |

**关键原则:** 在框架统一之前，**诚实标注每个结果所属框架**，避免混用。
