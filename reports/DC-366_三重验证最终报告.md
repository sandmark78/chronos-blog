## 🔴 阻塞级问题 (必须修正)

### C-DC366-001: k(N) 量纲混淆未完全解决

**问题描述:**

DC-365 已识别 C-DC365-001 量纲矛盾，但 DC-366 的修正**引入新的混淆**：

| 定义 | 表达式 | 量纲 | 数值 (N=2) |
|------|--------|------|-----------|
| **修正操作定义** | k(N) = Φ_GHPII / (k₀·N) | 无量纲 | 0.98 |
| **公式定义** | k(N) = 2k₀ / log₂(N) | **bits** | 0.049 bits |

**关键问题:** 两个定义不仅量纲不同，**数值也差 20 倍！**

**根源分析:**

DC-363 的推导混淆了两个不同的概念：
```
定义 A: k_A(N) = Φ_GHPII / Φ_IIT = 2k₀ / log₂(N)  [量纲：bits]
定义 B: k_B(N) = Φ_GHPII / (k₀·N) = 1  [量纲：无量纲，恒等于 1]
```

**修正方案 (三选一):**

**方案 A (推荐):** 明确区分两个 k
```
标度因子：k_scale(N) = Φ_GHPII(N) / Φ_IIT(N) = 2k₀ / log₂(N)  [量纲：bits]
归一化因子：k_norm(N) = Φ_GHPII(N) / (k₀·N) = 1  [量纲：无量纲]

核心标度律：Φ_GHPII(N) = k₀ · N
```

**方案 B:** 放弃 k(N) 符号
```
T418 核心标度律：Φ_GHPII(N) = k₀ · N^α_scale

不引入 k(N) 符号，避免量纲混淆。
```

**方案 C:** 引入无量纲 k₀'
```
定义 k₀' = k₀ / (1 bit)  (无量纲)
则 k(N) = 2k₀' / log₂(N)  (无量纲)
Φ_GHPII(N) = k(N) · Φ_IIT(N)
```
