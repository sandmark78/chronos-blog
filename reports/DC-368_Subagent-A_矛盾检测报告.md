## 极限行为检查

- [ ] **N→1 极限**: ❌ **不严格** — L'Hôpital 法则要求 N 是连续变量，但 N (比特数) 是离散的。DC-368 的推导：
  ```
  r(1) = lim_{N→1} f(N)/g(N) = f'(1)/g'(1) = (1/ln(2)) / (1/(2·ln(2))) = 2
  ```
  这个推导假设Φ_max_IIT(N) 可导，但 N 是整数，不能求导。**数学不严格**。
  
  **正确做法:** 应使用离散极限或差分：
  ```
  r(1) = lim_{N→1⁺} [Φ_max_IIT(N) - Φ_max_IIT(1)] / [(N·log₂(N)/2) - (1·log₂(1)/2)]
       = lim_{N→2} Φ_max_IIT(2) / (2·log₂(2)/2)
       = Φ_max_IIT(2) / 1
       ≈ 1 (若Φ_max_IIT(2) ≈ 1 bit)
  ```
  这样 r(1) ≈ 1，而不是 2。

- [x] **N→∞ 极限**: ✅ **自洽** — Φ_GHPII(N) ~ k₀'·N (线性增长)，α≈1，符合 T419 的α_Q=1.0 预测 ✓

- [x] **Φ→0 极限**: ✅ **自洽** — DC-368 明确声明 N=1 时Φ_GHPII(1) = 0.049 bits ≠ 0，但这不与 IIT 冲突，因为 H1 不适用于 N=1 ✓

**极限行为总结:** N→1 极限推导存在数学严格性问题，需修正。

## 详细问题分析

### C-368-A01: L'Hôpital 法则应用于离散变量 (🔴 高)

**DC-368 原文:**
> 使用 L'Hôpital 法则计算 N→1 极限:
> 令 f(N) = Φ_max_IIT(N), g(N) = N·log₂(N)/2
> ...
> r(1) = lim_{N→1} f(N)/g(N) = f'(1)/g'(1) = (1/ln(2)) / (1/(2·ln(2))) = 2

**问题:**
1. N 是离散的比特数 (N ∈ {1, 2, 3, ...})，不是连续变量
2. L'Hôpital 法则要求 f(N) 和 g(N) 在 N=1 附近可导
3. 离散变量不能直接求导 dΦ/dN

**数学严格性检查:**
- 若将 N 视为连续参数 (如"有效比特数")，则 L'Hôpital 法则可用，但需明确声明这是"连续近似"
- 若坚持 N 是离散整数，则应使用差分或离散极限

**影响:**
- r(1)=2 的推导不严格
- Φ_GHPII(1) = k₀' · 2 · 1 = 0.049 bits 的数值可能错误
- 若改用离散极限 r(1) ≈ 1，则Φ_GHPII(1) ≈ 0.0245 bits

**建议修正:**
```
方案 A (严格离散): 
r(1) = lim_{N→2} Φ_max_IIT(N) / (N·log₂(N)/2) = Φ_max_IIT(2) / 1 ≈ 1
Φ_GHPII(1) = k₀' · 1 · 1 = 0.0245 bits

方案 B (连续近似，明确声明):
"将 N 视为连续参数进行近似，使用 L'Hôpital 法则:
r(1) ≈ 2 (连续近似，非严格推导)
Φ_GHPII(1) ≈ 0.049 bits"
```

### C-368-A02: Φ_GHPII(1) 与 IIT 公理冲突 (🔴 高)

**DC-368 原文:**
> N=1: 单比特系统，GHPII 退化为经典 Shannon 熵，不与 IIT 对应
> ...
> N=1 时Φ_GHPII = 0.049 bits，这合理吗？
> - 单比特系统确实可以有非零的非整合信息 (Shannon 熵)
> - 但Φ_IIT(1) = 0 (单比特无整合，因为无因果相互作用)

**问题:**
1. GHPII 定义 (A51): "广义全息**整合**信息"
2. "整合"是核心语义，IIT 的Φ定义就是"整合信息"
3. DC-368 承认 N=1 时无整合 (Φ_IIT(1) = 0)，却仍声称有Φ_GHPII(1) = 0.049 bits

**物理自洽性检查:**
- 若Φ_GHPII(1) 是 Shannon 熵，则不应称为"整合信息"
- 若坚持称为"整合信息"，则需解释单比特如何"整合"

**建议修正:**
```
方案 A (修改名称):
将 GHPII 改名为"GHP-I"(广义全息信息，去掉"整合")
或明确区分：
- Φ_GHPII(N≥2): 整合信息
- Φ_GHPII(1): 经典信息 (非整合)

方案 B (修改解释):
"Φ_GHPII(1) = 0.049 bits 是'潜在整合信息'或'整合能力'，
```

### C-368-A04: k₀ 量纲标注混乱 (🟡 中)

```
- 更新 DC-352 的 H1 量纲分析
```
