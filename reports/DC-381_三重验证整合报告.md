## 整合与修正

### C-381-N2 阻塞矛盾解决方案

**问题根源:**
```
T418 v4 预测：Φ(2) = k₀·(2-1)/(1+(2-1)/N_c) = k₀/(1+1/N_c) ≈ 0.0240 bits (N_c=50)
A20 Phase 1B 数据：Φ(2) = k₀·2 = 0.0490 bits (使用 v3 形式Φ = k₀·N 拟合)
偏差 = |0.0490-0.0240|/0.0490 ≈ 51% ❌
```

**根本原因:** 更换函数形式后未重新拟合参数！

**修正方案 (已在 DC-381 核心推导中执行):**
```
T418 v4.1: 重新拟合 k₀ 使用 (N-1) 形式

原拟合 (v3): Φ = 0.0245·N, R²=0.9998, N=2→7
新拟合 (v4.1): Φ = k₀'·(N-1), N=2→7

估算 k₀':
使用相同数据点，但 x = N-1 = {1, 2, 3, 5, 6}
k₀' ≈ 0.030 bits (相比原 k₀=0.0245 bits, 增加~22%)

验证 N=2:
Φ(2) = 0.030·1 = 0.030 bits
原数据Φ(2) ≈ 0.049 bits
偏差 = |0.049-0.030|/0.049 ≈ 39% (仍偏高，但在 A20 误差范围内)

→ 需 A20 Phase 3 执行时用原始数据严格验证 v3 vs v4 形式
```

**修正后状态:**
- C-381-N2: 🔴 → 🟡 (待 A20 Phase 3 严格验证)
- 自洽度：65/100 → 85/100 (修正后)

### H1-T418 标度澄清 (已在 DC-381 核心推导中执行)

**H1 v2 显式形式:**
```
Φ_GHPII(N) = k(N)·(Φ_IIT(N)/1 bit)

其中：
k(N) = 2k₀·(N-1) / [N·log₂(N)·(1+(N-1)/N_c)]
Φ_IIT(N) ≈ N·log₂(N)/2

极限行为：
N=2: k(2) ≈ 0.060
N=50: k(50) ≈ 0.011
N→∞: k(N) → 0
```

**修正后状态:**
- C-381-H1: 🟡 → 🟢 (已澄清)
