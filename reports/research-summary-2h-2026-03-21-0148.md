## 9. 深度分析 — DC-366 量纲矛盾的根源与解决

### 9.1 问题根源 — 两个 k 的混淆

DC-363 的推导中，**无意中定义了两个不同的 k(N)**：

```
定义 A (标度因子): k_A(N) = Φ_GHPII(N) / Φ_IIT(N) = 2k₀ / log₂(N)  [量纲：bits]
定义 B (归一化因子): k_B(N) = Φ_GHPII(N) / (k₀·N) = 1  [量纲：无量纲，恒等于 1]
```

**DC-365 发现的问题:**
- 操作定义：k(N) = Φ_GHPII / Φ_IIT (无量纲比值)
- 公式定义：k(N) = 2k₀ / log₂(N) (bits 量纲)
- **量纲不一致!**

**DC-366 的"修正"引入的新问题:**
- 修正操作定义：k(N) = Φ_GHPII / (k₀·N) = 1 (无量纲)
- 公式定义维持：k(N) = 2k₀ / log₂(N) (bits)
- **数值差 20 倍!** (N=2: 0.98 vs 0.049)

### 9.2 解决方案对比

| 方案 | 核心思想 | 优点 | 缺点 | 推荐度 |
|------|---------|------|------|--------|
| **方案 A** | 明确区分 k_scale 和 k_norm | 物理意义清晰，保留两个概念 | 符号增加，需重新表述 T418 | ⭐⭐⭐⭐⭐ |
| **方案 B** | 放弃 k(N) 符号 | 简洁，避免混淆 | 丢失标度因子的直观表达 | ⭐⭐⭐ |
| **方案 C** | 引入无量纲 k₀' | 量纲自洽 | k₀ 物理意义模糊化 | ⭐⭐ |

**推荐方案 A:**
```
标度因子：k_scale(N) = Φ_GHPII(N) / Φ_IIT(N) = 2k₀ / log₂(N)  [量纲：bits]
归一化因子：k_norm(N) = Φ_GHPII(N) / (k₀·N) = 1  [量纲：无量纲]

核心标度律：Φ_GHPII(N) = k₀ · N  (N ≥ 2)
边界条件：Φ_GHPII(1) = 0
```

### 9.3 教训与改进

1. **量纲分析必须在定理推导初期进行** — 而非事后修正
2. **符号命名需清晰描述物理意义** — 避免 k(N) 等多义符号
3. **修正方案需彻底验证** — DC-365→DC-366 显示"修正"可能引入新问题
4. **Subagent-A 矛盾检测需提前介入** — 在定理推导过程中，而非完成后
