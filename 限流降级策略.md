## 🔧 降级策略

### 级别 1: 温和降级（当前触发）

**触发条件：** 单次 HTTP 500/502/503

**措施：**
```python
# 文献搜索降级
if error == "HTTP 500":
    延迟 5 分钟重试
    if 重试失败:
        降级为本地文献库分析
        标记任务为"部分完成"
```

**影响：** 文献搜索延迟 5-15 分钟

---

### 级别 2: 中度降级

**触发条件：** 连续 2 次失败 或 错误率 > 20%

**措施：**
```python
# 研究循环降级
if consecutive_failures >= 2:
    研究循环间隔：15 分钟 → 30 分钟
    文献搜索间隔：15 分钟 → 30 分钟
    启用本地缓存优先
```

**影响：** 产出减少 50%，但系统稳定

---

### 级别 3: 激进降级

**触发条件：** 连续 3 次失败 或 错误率 > 50%

**措施：**
```python
# 紧急降级
if consecutive_failures >= 3:
    研究循环间隔：30 分钟 → 60 分钟
    文献搜索暂停 1 小时
    仅执行核心任务 (研究循环)
    发送告警通知
```

**影响：** 产出减少 75%，保证系统不崩溃

---

### 级别 4: 生存模式

**触发条件：** Gateway 过载 或 Token 耗尽

**措施：**
```python
# 生存模式
if gateway_overloaded or token_exhausted:
    暂停所有自动任务
    仅响应人工指令
    保存当前状态
    发送紧急告警
```

**影响：** 系统暂停，等待恢复

## 📊 智能限流算法

### 动态间隔调整

```python
def adjust_interval(task_type, error_rate, success_rate):
    base_interval = get_base_interval(task_type)
    
    if error_rate > 0.5:
        return base_interval * 4  # 4x 间隔
    elif error_rate > 0.2:
        return base_interval * 2  # 2x 间隔
    elif error_rate > 0.1:
        return base_interval * 1.5  # 1.5x 间隔
    elif success_rate == 1.0:
        return max(base_interval * 0.8, min_interval)  # 逐步加速
    else:
        return base_interval  # 保持不变
```

### Token 预算管理

```python
# 每日 Token 预算
daily_budget = 50_000_000  # 50M tokens/日
current_usage = get_current_usage()

if current_usage > daily_budget * 0.8:
    # 80% 预算已用，降速
    reduce_interval(1.5x)
elif current_usage > daily_budget * 0.9:
    # 90% 预算已用，大幅降速
    reduce_interval(3x)
elif current_usage > daily_budget * 0.95:
    # 95% 预算已用，暂停非核心任务
    pause_non_critical_tasks()
```
